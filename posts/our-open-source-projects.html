Grafana Reporter

Grafana Reporter is a pure front-end Grafana app plugin that turns any dashboard into a PDF. It runs entirely in the browser using Grafana's official grafana-image-renderer — no backend service, no bundled Chromium, and no service accounts — so exports respect the signed-in user's datasource permissions and RBAC. It renders each panel through /render/d-solo and composes the result with jsPDF, resolving repeated panels and template variables (including $__all) along the way. Layout (panels per page, spacing, orientation), header/footer branding with a logo library, page numbers and custom text elements are all configurable — globally, via query parameters, or per run — and a report can be triggered from a dashboard link or a shareable URL.

cgstats

cgstats.sh is a single POSIX shell script that shows live CPU, memory and disk usage straight from Linux cgroups (v1/v2). It is built to run inside a container with zero dependencies — no agents, sidecars or Prometheus — and reports cores used versus the configured limit, memory versus its limit, and usage for arbitrary disk paths. Output is either colorized human-readable tables (with configurable warning/critical thresholds) or machine-readable JSON for dashboards and CI; it can refresh on an interval or print a single sample with --once. Install is a one-line curl, and it has been primarily developed and tested against cgroup v2.

ULID.jl

ULID.jl is a Julia implementation of ULIDs — Unique Lexicographically Sortable Identifiers, a 128-bit id combining a 48-bit timestamp with 80 bits of randomness. Unlike string-based implementations, it stores each ULID as a compact binary UInt128, which keeps them small and fast to compare and sort while staying time-ordered. You can generate fresh ids with ulid() or parse them from their canonical Crockford base32 text — handy as sortable keys in databases and distributed systems.

okd-nvidia-image

okd-nvidia-image builds an ostree-native container image for OKD 4.18–4.20 (SCOS, based on CentOS Stream 10) that bakes in the NVIDIA open kernel modules and userspace libraries. It layers packages onto the SCOS base with rpm-ostree — pinning the newest CS10 kernel available at build time, compiling the kmod-nvidia-open-dkms modules against that exact kernel ABI, and installing the CUDA driver libraries and tools like nvidia-smi — then commits the whole filesystem with ostree container commit. The kernel is pinned to the newest available rather than the node's running kernel because the installed kernel can be old enough that packages required to build DKMS modules are no longer available for it. The result is rolled out as the node OS image (e.g. via a MachineConfigPool osImageURL), giving OKD nodes reproducible GPU support.
